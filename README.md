Частина 1 — Картки розділу (блок ACF)

Блок Gutenberg з картками реалізовано на через ACF Blocks
Секція карток, що може бути використана повторно, реалізована як блок ACF на основі наданого дизайну Figma.

Враховуючи мету тестового завдання я сильно не витрачав час в реалізацію точної верстки дизайну Figma, так як метою є робота з ACF Blocks / ACF / Gutenberg та 
якості та структури проекту. Тому у верстці можуть бути незначні розбіжності.

ACF поля створив через код fuuction.php, щоб продемонструвати що можу працювати не тільки через інтерфейс ACF а і чирез код.

Основні характеристики:

- Реалізовано як блок ACF
- Повністю може бути використана повторно та налаштована з редактора
- Макет на основі сітки Bootstrap 5
- Використовує дизайн-токен з theme.json
- Стилі інтерфейсу та редактора повністю синхронізовані


Кожна картка підтримує:
Пресети ширини карток (сітка з 12 стовпцями):

4 / 12
6 / 12
8 / 12

Пресети типів карток:

Тільки текст
Текст + зображення (праворуч)
Текст + зображення (знизу)
Попередні налаштування кольору фону
(суворо обмежені кольорами з дизайну Figma)
Опціональна кнопка CTA

Технічні особливості:
CSS-змінні з theme.json використовуються для:
кольорів
відступів

Макет зображень використовує object-fit: cover
Картки однакової висоти за допомогою flex/grid

Реагування на мобільні пристрої
Стилі редактора (editor.css) точно віддзеркалюють макет інтерфейсу
CSS Bootstrap не завантажується в редактор (тільки стилі в межах області дії)

Частина 2 — Розділ Properties

Опис
Динамічний розділ з переліком Properties, розміщений у нижній частині головної сторінки.
Тип публікації: Properties

Підтримує:
- заголовок
- зображення

Сортування за ціною
Сортування запускається натисканням кнопки Sort by price (Low to High) 

Без випадаючих меню — натискання перемикає:
ASC → DESC
DESC → ASC

Візуальна стрілка та мітка оновлюються динамічно.
Відображається загальна кількість елементів.

Реалізація:
Сортування здійснюється за допомогою AJAX.
Запит сортується за допомогою meta_value_num.
Список перерендериться без перезавантаження сторінки.


Load more

Поведінка
Спочатку завантажується 5 елементів.

Якщо існує більше елементів:
Відображається кнопка Load more

Натискання кнопки:

Завантажує наступні 5 елементів.
Додає їх до списку.
Кнопка автоматично зникає, коли більше елементів немає.

Деталі реалізації:
Використовує параметр paged в WP_Query
Стан пагінації зберігається в JS

Архітектура JavaScript
Один файл JS: assets/js/properties.js

Делегування подій для:
сортування
завантаження додаткових даних

Централізований об'єкт стану:

{
  order: “asc”,
  page: 1
}

Використовує wp_localize_script для передачі URL-адреси admin-ajax.php.
Без вбудованого JS.
